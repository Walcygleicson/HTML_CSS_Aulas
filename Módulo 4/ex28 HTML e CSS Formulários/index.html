Usando Label

O campo de informação deve estar veinculado ao input para uma semantica melhor, para que isso aconteça usamos um label. Dentro da tag label, o parametro for="" pede para quem/ ou qual input ele está se referindo. Deve se colocar o nome dado ao input. Desta forma, o texto terá uma relação direta ao input referente a ele.

Métodos GET e POST

Porta padrão, o param method já vem com GET onde o local dos dados fica visível na url. Para esconder essa informação, colocaremos o POST

Envios de Formulários Vazios

Os campos de nome de usuário e senha sempre são obrigatorios e não podem ser enciados caso nenhuma infromação seja dada. Para que um formulário vazio nao seja enviado, usaermos o parametro required na tag de input assim tornando ela um campo obrigatório

Min e Max de caracteres

Podemos definir uma quantidade mínima e máxima de caracteres nos inputs com o parametro minlenth e maxlenth. Feito isto, se o usário digitar uma quantidade inferiro de caractere, uma caixa de diálogo parecerá alertando, e se o usuário tentar digitar um quantidade a mais que o max definido não irá conseguir

Placeholder

Podemos também colocar um placeholder, um texto de dica que ficará visivel dentro da caixa somente enquanto nada for digitado.

Tipos de Input

fieldset e legend

Envelopando os inputs comum fieldset podemos agrupar tudo em um campo com bordas padrão. Usando a tag legend é possível criar uma legenda.
